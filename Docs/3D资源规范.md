# 3D 资源规范

本文档定义了项目中 3D 资源的制作和导入规范，确保跨端兼容性和性能优化。

## 一、模型规范

### 1. 文件格式

- **推荐格式**：FBX
- **版本要求**：FBX 2018 或更高版本
- **不支持格式**：MAX、MAYA 原生格式（必须导出为 FBX）

### 2. 模型命名规范

```
类型_名称_版本.fbx
```

示例：
- `Prop_House_01.fbx`（房屋模型第1版）
- `Character_Car_01.fbx`（汽车棋子模型）
- `Building_Hospital_01.fbx`（医院建筑模型）

### 3. 模型要求

#### （1）面数限制

| 模型类型 | 最大面数 | 建议面数 |
|----------|----------|----------|
| 棋子模型 | 1000 | 500-800 |
| 房屋模型 | 2000 | 1000-1500 |
| 建筑模型 | 5000 | 2000-3000 |
| 棋盘模型 | 10000 | 5000-8000 |

#### （2）UV 展开

- 所有模型必须正确展开 UV
- UV 重叠率控制在合理范围
- 确保纹理映射正确

#### （3）材质设置

- 使用 PBR（Physically Based Rendering）材质
- 材质参数设置合理（Albedo、Metallic、Smoothness）
- 避免使用过多材质球（单个模型建议不超过 2 个材质）

#### （4）动画要求

- 动画片段命名规范：`动作名称_时长`
- 动画时长控制在合理范围（如：行走动画 1-2 秒）
- 动画曲线平滑，避免卡顿

### 4. 模型分类

#### （1）棋盘模型

- 尺寸：直径 5-6 米
- 格子数：36 格环形排列
- 高度：0.5-1 米（便于棋子移动）

#### （2）棋子模型

- 尺寸：高度 0.3-0.5 米
- 数量：4 个差异化模型
- 风格：卡通风格，色彩鲜明

#### （3）地产模型

- 房屋等级：1-3 级
- 1级：平房（高度 0.5 米）
- 2级：小楼（高度 1 米）
- 3级：高楼（高度 1.5 米）

#### （4）特殊场所模型

| 场所名称 | 模型特点 | 尺寸要求 |
|----------|----------|----------|
| 医院 | 病床元素，白色主色调 | 1.5×1.5×2 米 |
| 警局 | 牢房元素，蓝色主色调 | 1.5×1.5×2 米 |
| 银行 | 金库元素，金色主色调 | 1.5×1.5×2 米 |
| 税务局 | 文件元素，灰色主色调 | 1.5×1.5×2 米 |
| 公园 | 树木元素，绿色主色调 | 1.5×1.5×2 米 |
| 机会/命运 | 卡片元素，紫色主色调 | 1.5×1.5×2 米 |

#### （5）道具卡模型

- 尺寸：0.2×0.3×0.02 米
- 双面设计（正面图案，背面统一）
- 支持翻转动画

## 二、纹理规范

### 1. 文件格式

- **推荐格式**：PNG（支持透明通道）
- **颜色深度**：32 位（RGBA）
- **不支持格式**：JPG（不支持透明）、TGA（跨端兼容性差）

### 2. 纹理命名规范

```
模型名称_纹理类型.png
```

示例：
- `House_Albedo.png`（房屋漫反射纹理）
- `Car_Normal.png`（汽车法线纹理）
- `Hospital_Metallic.png`（医院金属度纹理）

### 3. 纹理分辨率

| 纹理类型 | 最大分辨率 | 建议分辨率 |
|----------|------------|------------|
| 主纹理（Albedo） | 2048×2048 | 1024×1024 |
| 法线纹理 | 2048×2048 | 1024×1024 |
| 金属度/光滑度 | 1024×1024 | 512×512 |
| 环境遮挡 | 1024×1024 | 512×512 |

### 4. 纹理要求

#### （1）Albedo 纹理

- 色彩鲜艳，对比度适中
- 支持透明通道（用于镂空效果）
- 避免纯黑色（影响光照计算）

#### （2）Normal 纹理

- 格式：OpenGL 格式
- 细节丰富，避免过度模糊
- 确保法线方向正确

#### （3）Metallic/Smoothness 纹理

- Metallic：0-1 范围（0=非金属，1=金属）
- Smoothness：0-1 范围（0=粗糙，1=光滑）
- 可以合并为一张纹理（Metallic 存放在 R 通道，Smoothness 存放在 A 通道）

### 5. 纹理分类

#### （1）棋子纹理

- 色彩鲜明，便于区分不同玩家
- 每个棋子使用独特的主色调
- 支持简单的反光效果

#### （2）地产纹理

- 不同颜色区分不同地产组
- 房屋等级通过纹理细节区分
- 支持凹凸效果（使用法线纹理）

#### （3）特殊场所纹理

- 每个场所使用独特的主题色
- 纹理细节突出场所特点
- 支持发光效果（使用 Emission 纹理）

#### （4）道具卡纹理

- 正面：卡片名称、图标、效果描述
- 背面：统一设计，带有游戏 logo
- 支持透明边框效果

## 三、动画规范

### 1. 文件格式

- 使用 Unity 原生 Animation Clip 格式
- 通过 FBX 导入动画

### 2. 动画命名规范

```
模型名称_动作名称_时长.anim
```

示例：
- `Dice_Roll_1.0.anim`（骰子滚动动画，时长 1 秒）
- `Car_Move_0.5.anim`（汽车移动动画，时长 0.5 秒）
- `Card_Flip_0.3.anim`（卡片翻转动画，时长 0.3 秒）

### 3. 动画类型

#### （1）基础动画

- **骰子滚动**：随机点数动画，时长 1-1.5 秒
- **棋子移动**：沿格子平滑移动，时长根据距离调整
- **卡片翻转**：3D 翻转动画，时长 0.3-0.5 秒

#### （2）交互动画

- **地产升级**：房屋生长动画，时长 0.5-1 秒
- **破产动画**：棋子倒地/消失动画，时长 1-2 秒
- **送医/送警**：场景切换动画，时长 0.5-1 秒

#### （3）UI 动画

- **按钮点击**：缩放+颜色变化，时长 0.1-0.2 秒
- **面板弹出**：淡入+缩放，时长 0.3-0.5 秒
- **提示框**：闪烁+颜色变化，时长 0.5-1 秒

### 4. 动画要求

- 动画曲线平滑，避免突然变化
- 循环动画设置为 Loop 模式
- 非循环动画设置为 Once 模式
- 动画速度适中，确保玩家能够看清

## 四、导入设置

### 1. 模型导入设置

#### （1）通用设置

- `Scale Factor`：1.0（保持原始尺寸）
- `Mesh Compression`：Low（平衡质量和性能）
- `Optimize Mesh`：勾选
- `Generate Colliders`：根据需要勾选

#### （2）动画设置

- `Import Animation`：勾选（如有动画）
- `Animation Type`：Generic（非人形动画）
- `Root Transform Position (Y)`：Bake Into Pose

### 2. 纹理导入设置

#### （1）通用设置

- `Texture Type`：根据用途选择
- `Compression`：Normal（跨端兼容）
- `Alpha Is Transparency`：勾选（如有透明通道）

#### （2）特定设置

- **Albedo**：Texture Type = Albedo
- **Normal**：Texture Type = Normal Map
- **Metallic/Smoothness**：Texture Type = Metallic Smoothness

### 3. 资源组织

- 按类型分类存放资源
- 使用文件夹结构管理：Models/、Textures/、Animations/
- 资源命名统一，便于查找和管理

## 五、性能优化

### 1. 模型优化

- 使用 LOD（Level of Detail）技术
- 合并静态物体（Static Batching）
- 降低不必要的细节

### 2. 纹理优化

- 使用合适的压缩格式
- 降低纹理分辨率
- 使用纹理图集（Texture Atlas）合并小纹理

### 3. 动画优化

- 使用动画压缩（Animation Compression）
- 减少关键帧数量
- 复用动画片段

### 4. 加载优化

- 使用资源异步加载
- 预加载常用资源
- 使用对象池管理频繁创建的对象

## 六、跨端兼容性

### 1. 格式兼容性

- 确保所有资源使用跨端兼容格式
- 避免使用平台专属功能

### 2. 性能适配

- 针对不同性能的设备优化资源
- 提供低/中/高画质选项

### 3. 测试要求

- 在 Mac 和 Windows 平台分别测试资源加载
- 确保资源在不同分辨率下显示正常
- 检查动画在不同平台播放是否流畅

---

遵循以上规范，确保 3D 资源在跨端环境下能够正常加载和显示，同时保持良好的性能表现。
