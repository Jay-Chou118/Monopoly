# 跨端配置指南

本文档详细介绍如何配置 Unity 工程，实现 Mac/Windows 跨平台打包。

## 一、Unity 版本要求

推荐使用 Unity 2020.3 LTS 或更高版本，确保跨端兼容性。

## 二、项目设置配置

### 1. 打开项目设置

在 Unity 编辑器中，依次点击：`Edit` → `Project Settings`

### 2. Player 设置

#### （1）通用设置

- **Company Name**：设置公司名称（如：MonopolyStudio）
- **Product Name**：设置产品名称（如：Monopoly 3D）
- **Version**：设置版本号（如：1.0.0）

#### （2）分辨率与演示

- **Default Screen Width**：1920
- **Default Screen Height**：1080
- **Fullscreen Mode**：Windowed（开发阶段）/ Fullscreen（发布阶段）

#### （3）优化设置

- **Optimize Frame Pacing**：勾选
- **VSync Count**：Every V Blank

### 3. 平台特定设置

#### （1）Windows 设置

1. 在 Platform 列表中选择 `PC, Mac & Linux Standalone`
2. 点击 `Switch Platform`（首次切换时需要等待）
3. 设置 `Target Platform` 为 `Windows`
4. 设置 `Architecture` 为 `x86_64`（64位）

#### （2）macOS 设置

1. 在 Platform 列表中选择 `PC, Mac & Linux Standalone`
2. 点击 `Switch Platform`（首次切换时需要等待）
3. 设置 `Target Platform` 为 `macOS`
4. 设置 `Architecture` 为 `Universal`（支持 Intel 和 Apple Silicon）

### 4. 输入设置

- 确保使用 Unity 新输入系统（Input System Package）
- 配置键鼠和手柄输入映射

## 三、资源管理配置

### 1. 模型格式要求

- 所有 3D 模型使用 FBX 格式
- 模型导入设置：
  - `Mesh Compression`：Low（平衡质量和性能）
  - `Optimize Mesh`：勾选
  - `Generate Colliders`：根据需要勾选

### 2. 纹理格式要求

- 所有纹理使用 PNG 格式
- 纹理导入设置：
  - `Texture Type`：根据用途选择（如：Albedo, Normal Map）
  - `Compression`：Normal（跨端兼容）
  - `Alpha Is Transparency`：勾选（如有透明通道）

### 3. 动画设置

- 使用 Unity 原生 Animation Clip 格式
- 确保动画片段命名规范

## 四、构建配置

### 1. Windows 构建配置

1. 打开 `File` → `Build Settings`
2. 添加需要构建的场景（拖拽场景到 Scenes In Build 列表）
3. 设置 `Target Platform` 为 `Windows`
4. 设置 `Architecture` 为 `x86_64`
5. 点击 `Player Settings` 进行最终检查
6. 点击 `Build`，选择输出目录为 `Monopoly_Unity_Project/Build/Windows`

### 2. macOS 构建配置

1. 打开 `File` → `Build Settings`
2. 添加需要构建的场景（拖拽场景到 Scenes In Build 列表）
3. 设置 `Target Platform` 为 `macOS`
4. 设置 `Architecture` 为 `Universal`
5. 点击 `Player Settings` 进行最终检查
6. 点击 `Build`，选择输出目录为 `Monopoly_Unity_Project/Build/macOS`

## 五、跨端测试流程

### 1. Windows 测试

1. 在 Windows 系统上运行构建的 `.exe` 文件
2. 测试核心功能：
   - 游戏启动是否正常
   - 棋子移动是否流畅
   - 按钮点击是否响应
   - 动画播放是否正常

### 2. macOS 测试

1. 在 macOS 系统上运行构建的 `.app` 文件
2. 测试核心功能：
   - 游戏启动是否正常（注意权限设置）
   - 棋子移动是否流畅
   - 按钮点击是否响应
   - 动画播放是否正常

### 3. 兼容性检查

- 确保在不同分辨率下 UI 布局正常
- 确保音效和音乐播放正常
- 确保 3D 模型和纹理加载正常

## 六、常见问题解决

### 1. 构建失败

- 检查场景是否正确添加到 Build Settings
- 检查脚本是否有编译错误
- 检查资源路径是否正确

### 2. 运行时崩溃

- 检查空引用异常（NullReferenceException）
- 检查数组越界问题
- 检查资源加载失败

### 3. 跨端差异

- 避免使用平台专属 API
- 使用 Unity 跨平台 API
- 测试时注意不同平台的输入差异

### 4. macOS 权限问题

- 在 macOS 系统偏好设置中允许应用运行
- 检查文件访问权限

## 七、性能优化建议

### 1. 模型优化

- 降低模型面数（使用 LOD 技术）
- 合并静态物体（Static Batching）

### 2. 纹理优化

- 使用合适的压缩格式
- 降低纹理分辨率（如：2048x2048 → 1024x1024）

### 3. 代码优化

- 使用对象池减少 GC
- 优化 Update 方法中的计算
- 使用异步加载资源

### 4. 渲染优化

- 降低渲染批次
- 使用合适的光照设置
- 关闭不必要的特效

## 八、版本控制建议

1. 使用 Git 进行版本控制
2. 忽略 Unity 自动生成的文件（如：Library/）
3. 按功能模块创建分支
4. 定期合并稳定代码到 main 分支

---

通过以上配置，确保项目能够在 Windows 和 macOS 平台上正常构建和运行，实现真正的跨端兼容。
