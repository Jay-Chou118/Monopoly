# 《大富翁》3D 跨端（Mac/Windows）单机多人游戏开发策划文档

## 文档信息

### 项目名称
大富翁（Monopoly）3D 跨端版

### 核心目标
实现跨 Mac/Windows 平台的 3D 本地单机多人游戏，先完成核心玩法，再迭代 3D 视觉与跨端适配，后期扩展线上联机

### 开发阶段
- 第一阶段：核心逻辑 + 跨端基础
- 第二阶段：3D 视觉 + UI 优化
- 第三阶段：线上联机扩展

### 目标平台
Windows 10+、macOS 10.15+（Intel/Apple Silicon）

### 技术栈选型
- 优先推荐：Unity（跨端适配成熟、3D 生态完善、支持 Mac/Windows 一键打包）
- 备选：Unreal Engine（3D 质感更强，跨端配置稍复杂）

---

## 一、项目核心定位与范围

### 1. 核心玩法锁定（第一阶段必做）

**游玩模式**：本地单机多人（2~4 人同屏轮流操作，无需网络）

**核心流程**：掷骰子 → 棋子 3D 移动 → 触发格子事件（买地/收费/特殊场所/道具卡）→ 回合切换

**胜利条件**：所有对手破产（金币为负且无抵押地产），剩余最后1名玩家获胜

**不做范围（第一阶段）**：线上联机、AI 对手、存档/读档、排行榜（后期扩展）

### 2. 跨端与 3D 核心要求

**跨端**：同一套代码/工程，可直接编译为 Windows 客户端（.exe）和 macOS 客户端（.app），无需大幅修改逻辑

**3D 效果**：基础 3D 棋盘（立体地块、棋子）、简单动画（骰子滚动、棋子行走、卡片翻开）、光影氛围（基础光照，无需高端特效）

**UI 规划**：先实现逻辑可用的 3D 交互 UI（如按钮、面板、提示框），后续再迭代精致美术 UI

---

## 二、核心系统设计（跨端兼容版）

### 1. 棋盘与格子系统（3D 适配）

#### （1）棋盘规格

**总格子数**：36 格（平衡玩法与 3D 展示），采用环形布局

**格子类型分布**：

| 格子类型 | 数量 | 核心功能 |
|----------|------|----------|
| 起点 | 1 | 绕圈一圈奖励 500 金币 |
| 普通地产 | 24 | 可购买、升级（1~3 级房屋）、收取过路费 |
| 特殊场所 | 6 | 医院、警局、银行、税务局、公园、机会/命运 |
| 道具卡格 | 4 | 随机获得道具卡 |

#### （2）3D 格子设计要求

- **地产格**：立体房屋模型（1级=平房、2级=小楼、3级=高楼），不同地产用不同颜色区分
- **特殊场所**：专属 3D 模型（医院=病床模型、警局=牢房模型、银行=金库模型），触发时播放对应音效
- **跨端适配**：模型格式统一为 FBX（Unity 原生支持），纹理格式为 PNG（透明背景），确保 Mac/Windows 均能正常加载

### 2. 特殊场所规则（可开发落地版）

| 场所名称 | 触发条件 | 3D 交互效果 | 核心规则 |
|----------|----------|--------------|----------|
| 医院 | 被道具卡送入/命运事件 | 棋子进入医院模型，播放“病床提示动画” | 停留1回合，支付 200 金币；可使用“出院卡”立即离开 |
| 警局 | 命运事件/使用欺诈卡 | 棋子进入警局模型，弹出“保释选择面板” | 二选一：交 300 保释金立即离开，或停留1回合 |
| 银行 | 主动进入 | 弹出 3D 银行面板 | 抵押地产换金币、赎回地产；前期无贷款 |
| 税务局 | 踩到/绕圈一圈 | 弹出“税务扣除提示” | 支付当前现金 10%，持有“免税卡”豁免 |
| 公园 | 主动进入 | 棋子在公园停留，播放“休息动画” | 无收益/惩罚，直接结束回合 |
| 机会/命运 | 踩到 | 随机抽取事件卡（3D 卡片翻转动画） | 随机奖励/惩罚（金币增减、传送、送医/送警） |

### 3. 道具卡系统（3D 交互适配）

#### （1）道具卡清单（8 张，核心必做）

| 卡片类型 | 卡片名称 | 3D 交互 | 核心效果 |
|----------|----------|----------|----------|
| 功能卡 | 定点骰子卡 | 卡片翻开，显示骰子点数选择按钮 | 本回合指定掷出 1~6 任意点数 |
| 功能卡 | 双倍租金卡 | 卡片覆盖在目标地产上 | 下一次收取过路费翻倍 |
| 功能卡 | 免罚卡 | 卡片悬浮在玩家头顶 | 免疫一次医院/警局/税务惩罚 |
| 攻击卡 | 陷害卡 | 卡片指向目标玩家，触发送行动画 | 将指定玩家送入医院 |
| 攻击卡 | 拆迁卡 | 卡片点击目标地产，播放拆除动画 | 拆除对手1级房屋，支付少量补偿 |
| 攻击卡 | 勒索卡 | 卡片弹出金额输入框 | 抽取对手 100~300 金币 |
| 防御卡 | 出院卡 | 卡片翻转，播放治愈动画 | 自身在医院时立即离开 |
| 防御卡 | 保释卡 | 卡片翻转，播放解锁动画 | 自身在警局时立即离开 |

#### （2）道具卡规则

- **获取**：踩到道具卡格随机获得1张
- **持有上限**：每人 6 张（3D UI 展示为卡片栏）
- **使用限制**：仅自己回合可使用，使用后卡片消失

### 4. 经济系统（跨端数据兼容）

**初始金币**：每人 3000 金币（跨端统一配置，存为全局常量）

**核心经济规则**：

- 起点奖励：每绕地图一圈，获得 500 金币
- 地产过路费：随地产等级提升而增加，3级地产过路费最高
- 破产判定：金币为负，且无地产可抵押 → 立即破产，从游戏中移除
- 跨端数据：所有经济数据（金币、地产、道具）存储在内存中，关闭游戏即重置（前期不做存档）

### 5. 回合与玩家系统（本地多人）

**玩家管理**：支持 2~4 名玩家，可输入自定义昵称（3D 输入框 UI）

**回合流程**：

1. 当前玩家高亮显示（3D 头顶标识/边框变色）
2. 点击「掷骰子」按钮 → 播放骰子滚动 3D 动画
3. 棋子根据点数沿棋盘 3D 路径移动
4. 触发格子事件，弹出对应 3D 提示面板（如“是否购买该地产？”）
5. 完成所有操作后，点击「结束回合」，切换至下一位玩家

**跨端适配**：玩家数据（昵称、位置、金币）实时同步，确保 Mac/Windows 端数据逻辑一致

## 三、跨端与 3D 技术方案

### 1. 技术栈选型与跨端适配

| 选型 | 优势 | 跨端配置要点 |
|------|------|--------------|
| Unity（首选） | 1. 原生支持 Mac/Windows 跨平台打包；2. 完善的 3D 生态（模型、动画、光照）；3. 一键导出客户端；4. 脚本跨端兼容（C#） | 1. 工程设置中开启 Windows/macOS 构建支持；2. 避免使用平台专属 API；3. 资源统一为跨端格式（FBX/PNG）；4. 打包时勾选“通用二进制”（支持 Apple Silicon） |
| Unreal Engine（备选） | 1. 3D 质感更强，光影效果更逼真；2. 跨端配置成熟 | 1. 安装对应平台构建工具（Windows SDK、Xcode）；2. 蓝图/蓝图 C++ 跨端兼容；3. 资源导入时转换为跨端兼容格式 |

### 2. 3D 核心实现要求

**模型资源**：

- 棋盘：大尺寸立体棋盘，支持 36 格环形排列，可缩放/旋转视角（玩家可自由拖动查看棋盘）
- 棋子：4 个差异化 3D 模型（如汽车、飞机、机器人、动物），区分不同玩家
- 道具：卡片采用立体翻转模型，房屋采用分层立体模型

**动画系统**：

- 基础动画：骰子滚动（随机点数动画）、棋子行走（沿格子平滑移动）、卡片翻开/收起
- 交互动画：地产升级（房屋生长动画）、破产（棋子倒地/消失动画）、送医/送警（场景切换动画）
- 跨端兼容：动画格式为 Unity 原生 Animation Clip，确保 Mac/Windows 端播放流畅

**光影与氛围**：

- 基础光照：场景添加平行光+环境光，确保地块/模型轮廓清晰
- 简单特效：骰子滚动粒子特效、金币收集飘字特效、卡片发光特效
- 性能优化：降低模型面数、压缩纹理，确保 Mac/Windows 低端设备也能流畅运行

### 3. 跨端打包规范

**Windows 端**：打包为 64 位 .exe 安装包，包含运行依赖库（确保无额外运行环境即可运行）

**macOS 端**：打包为 .app 应用程序，支持 Universal 架构（兼容 Intel 与 Apple Silicon 芯片）

**打包流程**：

1. 完成核心逻辑开发后，在 Unity 中切换平台（Windows/macOS）
2. 配置打包参数（应用名称、图标、版本号）
3. 一键构建，生成对应平台客户端
4. 测试跨端兼容性（确保无崩溃、资源加载正常）

## 五、开发优先级（分阶段落地）

### 第一阶段：核心逻辑 + 跨端基础（1~2 周）

1. 搭建 Unity 工程，配置 Mac/Windows 跨端构建环境
2. 实现核心逻辑：玩家管理、回合系统、骰子移动、地产买卖、经济规则
3. 完成基础 3D 模型导入（棋盘、简单棋子、地产格）
4. 测试跨端编译：分别打包 Windows/macOS 客户端，验证核心逻辑无差异
5. 实现基础 3D 交互：棋子移动、骰子滚动动画

### 第二阶段：3D 视觉完善 + 特殊场所/道具卡（2~3 周）

1. 完善 3D 资源：特殊场所模型、道具卡模型、房屋升级模型
2. 实现特殊场所逻辑（医院、警局、银行等）+ 3D 交互动画
3. 完成道具卡系统（获取、使用、效果）+ 3D 卡片翻转动画
4. 实现基础 3D UI：主菜单、玩家面板、道具卡栏、提示弹窗
5. 优化跨端性能：压缩资源、调整光照，确保 Mac/Windows 流畅运行

### 第三阶段：本地多人完整体验 + 优化（1 周）

1. 实现 2~4 人本地多人完整流程（昵称输入、回合切换、破产淘汰）
2. 完善 UI 交互：按钮点击、面板显示/隐藏、输入框逻辑
3. 测试跨端兼容性：在 Mac/Windows 端分别测试，修复崩溃/资源加载问题
4. 优化体验：调整动画速度、UI 布局，确保操作流畅

### 第四阶段：后期扩展（不影响前期代码）

1. 扩展线上联机（新增网络模块，分离本地/联机逻辑）
2. 迭代精致 3D 美术 UI（替换资源、优化光影、添加特效）
3. 新增 AI 对手、存档/读档、积分模式

## 七、关键注意事项

1. **跨端兼容**：全程避免使用平台专属 API（如 Windows 的 Win32 接口、macOS 的 Objective-C），仅使用 Unity 原生接口/C# 通用代码
2. **资源管理**：3D 模型/纹理统一使用跨端格式，避免使用特殊插件或格式，确保 Mac/Windows 端正常加载
3. **逻辑解耦**：核心逻辑（经济、棋盘、道具）与 UI、3D 表现分离，便于后续独立迭代
4. **版本控制**：按模块创建分支（dev/core、dev/3d、dev/ui），main 分支合并稳定版本，避免代码冲突
5. **测试同步**：每完成一个模块，需在 Mac/Windows 端分别测试，确保逻辑一致、无崩溃
