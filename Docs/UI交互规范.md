# UI 交互规范

本文档定义了游戏中 UI 界面的设计和交互规范，确保用户体验的一致性和流畅性。

## 一、UI 设计原则

### 1. 一致性原则

- **视觉风格统一**：所有 UI 元素使用统一的设计语言
- **交互逻辑一致**：相似功能使用相同的交互方式
- **反馈机制统一**：操作反馈方式保持一致

### 2. 易用性原则

- **操作简单**：避免复杂的操作流程
- **反馈及时**：用户操作后立即给予视觉和音效反馈
- **容错性**：允许用户撤销错误操作

### 3. 美观性原则

- **色彩搭配**：使用协调的色彩方案
- **布局合理**：信息层次清晰，重点突出
- **动画流畅**：过渡动画自然，不影响操作

## 二、UI 组件规范

### 1. 按钮（Button）

#### （1）状态设计

- **正常状态**：默认显示样式
- **悬停状态**：鼠标悬停时的样式（如：颜色变化、缩放）
- **点击状态**：按下时的样式（如：缩放、颜色加深）
- **禁用状态**：不可点击时的样式（如：透明度降低）

#### （2）交互反馈

- 点击时播放音效
- 状态变化时有过渡动画
- 重要按钮有震动反馈

#### （3）命名规范

```
Button_功能名称
```

示例：
- `Button_RollDice`（掷骰子按钮）
- `Button_EndTurn`（结束回合按钮）
- `Button_BuyLand`（购买地产按钮）

### 2. 面板（Panel）

#### （1）类型分类

- **信息面板**：显示游戏信息（如：玩家金币、位置）
- **操作面板**：提供操作选项（如：购买、升级、抵押）
- **提示面板**：显示提示信息（如：事件描述、错误提示）

#### （2）显示/隐藏动画

- **显示动画**：淡入 + 缩放（时长 0.3-0.5 秒）
- **隐藏动画**：淡出 + 缩放（时长 0.3-0.5 秒）
- **过渡效果**：使用缓动函数确保动画流畅

#### （3）布局要求

- 面板居中显示
- 信息层次清晰
- 按钮排列合理

### 3. 输入框（InputField）

#### （1）类型分类

- **文本输入**：用于输入玩家昵称
- **数字输入**：用于输入金额、数量等

#### （2）交互设计

- 输入时实时验证
- 输入完成后自动提交
- 支持键盘导航（Tab 切换）

#### （3）限制条件

- 昵称长度：3-12 个字符
- 数字输入：限制最小值和最大值
- 禁止输入特殊字符

### 4. 提示框（Popup）

#### （1）类型分类

- **确认提示**：需要用户确认的操作（如：购买确认）
- **信息提示**：显示游戏信息（如：事件结果）
- **错误提示**：显示错误信息（如：金币不足）

#### （2）设计要求

- 背景半透明，突出提示内容
- 按钮布局合理（确认/取消）
- 自动关闭（可选）

### 5. 道具卡栏（CardBar）

#### （1）设计要求

- 水平排列，最多显示 6 张卡片
- 卡片大小：80×120 像素
- 选中状态：边框高亮

#### （2）交互设计

- 鼠标悬停：显示卡片详情
- 点击：选中卡片，显示使用选项
- 拖拽：支持拖拽使用（可选）

### 6. 玩家面板（PlayerPanel）

#### （1）显示内容

- 玩家昵称
- 当前金币数量
- 当前位置
- 道具卡数量
- 地产数量

#### （2）高亮效果

- 当前玩家：边框变色或发光效果
- 破产玩家：灰色显示或标记

## 三、游戏流程 UI 交互

### 1. 主菜单

#### （1）界面元素

- 游戏标题（3D 立体效果）
- 开始游戏按钮
- 设置按钮
- 退出按钮

#### （2）交互流程

1. 点击「开始游戏」→ 进入玩家设置界面
2. 点击「设置」→ 打开设置面板
3. 点击「退出」→ 退出游戏

### 2. 玩家设置

#### （1）界面元素

- 玩家数量选择（2-4 人）
- 玩家昵称输入框
- 开始游戏按钮
- 返回按钮

#### （2）交互流程

1. 选择玩家数量
2. 输入玩家昵称
3. 点击「开始游戏」→ 进入游戏场景

### 3. 游戏主界面

#### （1）界面元素

- 棋盘（3D 显示）
- 玩家面板（显示玩家信息）
- 道具卡栏（显示玩家持有的卡片）
- 操作按钮（掷骰子、结束回合）
- 当前回合提示

#### （2）回合流程交互

1. 当前玩家高亮显示
2. 点击「掷骰子」→ 播放骰子动画 → 棋子移动 → 触发格子事件
3. 处理格子事件（如：购买地产、支付过路费）
4. 点击「结束回合」→ 切换到下一位玩家

### 4. 格子事件交互

#### （1）普通地产

- 未购买：显示「购买」和「放弃」按钮
- 已购买：显示过路费信息

#### （2）特殊场所

- 医院：显示停留回合和支付金额，提供「使用出院卡」选项
- 警局：显示保释选项（支付保释金或停留）
- 银行：显示抵押和赎回选项
- 税务局：显示税务金额，提供「使用免税卡」选项
- 公园：显示休息动画，直接结束回合
- 机会/命运：播放卡片翻转动画，显示事件结果

#### （3）道具卡格

- 随机获得道具卡，显示卡片信息和动画

### 5. 道具卡使用交互

#### （1）使用流程

1. 点击道具卡栏中的卡片
2. 根据卡片类型显示使用选项
3. 选择目标（如：玩家、地产）
4. 播放卡片效果动画
5. 卡片从手中移除

#### （2）特殊卡片交互

- **定点骰子卡**：显示骰子点数选择按钮（1-6）
- **双倍租金卡**：选择目标地产，卡片覆盖在地产上
- **免罚卡**：直接使用，卡片悬浮在玩家头顶
- **陷害卡**：选择目标玩家，触发送行动画
- **拆迁卡**：选择目标地产，播放拆除动画
- **勒索卡**：弹出金额输入框（100-300）
- **出院卡**：在医院时自动使用
- **保释卡**：在警局时自动使用

## 四、动画与特效

### 1. 过渡动画

- 场景切换：淡入淡出效果（时长 0.5-1 秒）
- 面板切换：滑动或缩放效果（时长 0.3-0.5 秒）
- 卡片翻转：3D 翻转动画（时长 0.3-0.5 秒）

### 2. 反馈特效

- 金币变化：数字飘字特效（向上浮动，逐渐消失）
- 按钮点击：缩放+颜色变化（时长 0.1-0.2 秒）
- 卡片使用：发光特效（持续 1-2 秒）

### 3. 音效设计

- 按钮点击：短促的音效（0.1-0.2 秒）
- 骰子滚动：持续的滚动音效（1-1.5 秒）
- 卡片翻转：清脆的翻转音效（0.1-0.2 秒）
- 金币变化：金币音效（0.2-0.3 秒）

## 五、响应式设计

### 1. 分辨率适配

- 支持不同分辨率（1920×1080、1366×768 等）
- UI 元素自适应缩放
- 布局调整以适应不同屏幕比例

### 2. 输入适配

- 支持键鼠操作
- 支持手柄操作（可选）
- 触摸操作优化（可选）

## 六、性能优化

### 1. 渲染优化

- 使用 UI 图集（Sprite Atlas）合并小图标
- 减少 Draw Call
- 使用合适的材质和着色器

### 2. 内存优化

- 复用 UI 预制体
- 动态加载和卸载 UI 资源
- 避免内存泄漏

### 3. 交互优化

- 减少不必要的动画
- 优化事件响应
- 避免卡顿

## 七、测试要求

### 1. 功能测试

- 所有按钮点击正常
- 输入框验证正确
- 面板显示/隐藏正常

### 2. 交互测试

- 动画播放流畅
- 音效同步正常
- 反馈及时

### 3. 兼容性测试

- 在不同分辨率下显示正常
- 在不同平台（Mac/Windows）下表现一致
- 在不同性能设备上运行流畅

---

遵循以上规范，确保游戏 UI 界面设计美观、交互流畅，提供良好的用户体验。
